Fichier: anomalies-consolidated.controller.ts (NOUVEAU)

    - Total anomalies
    - Compteurs par sévérité (critical, warning)
    - Détails par type d'anomalie

      "mlDetected": { ... },

    }
  }


✅ 3. ANALYSE DES FLUX AMELIOREE

Fichier: material-flow.service.ts
Méthode: analyzeFlowAnomalies()

Fonctionnalités:
  • Calcul ratio entrées/sorties par matériau
  • Détection sorties excessives (>150% entrées)
  • Classification risques:
    - CRITICAL: sorties > 200% entrées
    - HIGH: sorties > 150% entrées
    - MEDIUM: anomalies mineures
    - LOW: normal
  
  • Messages d'alerte détaillés:
    "🚨 RISQUE CRITIQUE DE VOL/GASPILLAGE: 
     Sorties (250) sont 250% supérieures aux entrées (100)"
  
  • Identification chantiers à risque
  • Statistiques par site

Endpoint:
  GET /api/material-flow/analyze-anomalies?siteId=...&days=30

    FRONTEND - MODIFICATIONS RECOMMANDEES
═══════════════════════════════════════════════════════════════

Fichier: apps/frontend/src/app/pages/materials/AnomaliesList.tsx

1. UTILISER ENDPOINT CONSOLIDE
   const response = await fetch(
     '/api/materials/anomalies/consolidated?days=30'
   );
   const data = await response.json();

2. AFFICHER ANOMALIES DE FLUX
   {data.anomalies.flowAnalysis.map(site => (
     <div key={site.siteId}>
       <h3>{site.siteName}</h3>
       {site.materials.map(material => (
         <div className={getRiskClass(material.riskLevel)}>
           <p>{material.materialName}</p>
           <p>{material.riskReason}</p>
           <p>Entrées: {material.totalIn} | Sorties: {material.totalOut}</p>
         </div>
       ))}
     </div>
   ))}

3. AMELIORER AFFICHAGE EXPIRANTS
   {data.anomalies.expiringMaterials.map(material => (
     <div className={getSeverityClass(material.severity)}>
       <p>{material.materialName} ({material.materialCode})</p>
       <p>Expire dans {material.daysToExpiry} jours</p>
       <p>Stock: {material.currentStock}</p>
     </div>
   ))}
