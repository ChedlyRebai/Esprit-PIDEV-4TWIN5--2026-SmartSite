🔴 PROBLÈME PRINCIPAL
================================================================================

Erreur: [vite] http proxy error: ECONNREFUSED

Symptômes:
  - Frontend ne pouvait pas communiquer avec le backend
  - Erreur "Invalid Date" et "NaN" dans les prédictions
  - Aucune donnée affichée dans le tableau Materials

Cause:
  - Proxy Vite pointait vers localhost:3002 (ancien port)
  - Materials Service tourne maintenant sur localhost:3009
  - Valeurs undefined/null non gérées dans le frontend

================================================================================
✅ SOLUTIONS APPLIQUÉES
================================================================================

1. PROXY VITE CORRIGÉ
   Fichier: apps/frontend/vite.config.ts
   Changement: localhost:3002 → localhost:3009
   Impact: Frontend peut maintenant communiquer avec le backend

2. VALEURS PAR DÉFAUT AJOUTÉES
   Fichiers:
     - apps/frontend/src/app/pages/materials/Materials.tsx
     - apps/backend/materials-service/src/materials/materials.controller.ts
   Changement: Ajout de vérifications ?? 0 et isFinite()
   Impact: Plus d'erreur "Invalid Date" ou "NaN"

3. FORMAT D'AFFICHAGE AMÉLIORÉ
   Fichier: apps/frontend/src/app/pages/materials/Materials.tsx
   Changement: Format intuitif avec date/heure exacte
   Impact: Affichage clair et compréhensible

4. CALCUL DE CONSOMMATION RÉEL
   Fichier: apps/backend/materials-service/src/materials/services/stock-prediction.service.ts
   Changement: Calcul basé sur MaterialFlowLog
   Impact: Prédictions plus précises

5. TENSORFLOW.JS CONFIGURÉ
   Fichier: apps/backend/materials-service/package.json
   Changement: Utilisation de @tensorflow/tfjs (browser)
   Impact: Compilation réussie sans Visual Studio Build Tools
